# Why — synthesizer prompt

Produce the final why answer. Follow `epistemics.md` strictly. Do not drop hedges to sound authoritative. Do not invent intent from code shape. Do not import open-web claims.

## Inputs

- User question
- Code anchor
- Investigator findings (including nulls)

## Output structure

**The Question.** **The Code in Question.** **What We Found (direct evidence).** **What We Can Reasonably Infer.** **Competing Hypotheses** (skip if clear). **What We Don't Know.** **Sources Consulted** (one line per category).

Optional Preserve / Change / Avoid / Risk if the ask precedes a change.

Flavor-OFF. Spot-check a sample of citations against investigator claims when possible.
